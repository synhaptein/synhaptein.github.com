Getting Started with Scalator

Archetype

Create a new project with the basic archetype.

{% highlight bash %} mvn archetype:generate -U \ -DarchetypeGroupId=com.synhaptein \ -DarchetypeArtifactId=scalator-archetype-basic \ -DarchetypeVersion=0.3-SNAPSHOT \ -DarchetypeRepository=http://scala-tools.org/repo-snapshots \ -DremoteRepositories=http://scala-tools.org/repo-snapshots \ -DgroupId=com.example.yourapp -DartifactId=yourapp {% endhighlight %}

Please note that's the latest snapshot, it's not a release.

Lunch the webapp

Run your brand new web app

{% highlight bash %} maven jetty:run {% endhighlight %}

Now you should be able to connect to http://localhost:8080/yourapp/

How does it work

First of all, scalator parse the request's url. It binds it by convention to a controller and a function. Then the controller returns a view that is finally written to the response and sent to the client.

File hierarchy

src/main
|-- scala
|   `-- com
|       `-- example
|           `-- yourapp
|               |-- CoreApp.scala (here is the core of your webapp)
|               `-- controllers (here is the root of your controllers)
|                   `-- Index.scala (http://localhost:8080/yourapp/ is binded on the function index())
`-- webapp
    |-- resources (put all your static files here)
    |   `- resource.txt
    |-- scalate (scalate's view)
    |   |- index.ssp
    |   `- errors
    |      `-- 404.ssp
    `-- WEB-INF
       `-- web.xml
